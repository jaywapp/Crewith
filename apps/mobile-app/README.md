# Crewith Mobile App

Flutter 기반 회원 앱과 운영진 앱 모드의 기본 구조입니다.

현재 Flutter CLI가 로컬에서 응답하지 않아 `flutter create` 대신 최소 프로젝트 파일을 먼저 작성했습니다. Flutter CLI 문제를 해결한 뒤 `flutter pub get`과 Android 플랫폼 생성을 검증해야 합니다.

---

## 수동 테스트 스크립트

처음 보는 개발자가 회원 앱의 주요 기능을 확인할 수 있도록 절차와 기대 결과를 정리한다.

### 사전 준비

#### 1. API 서버 실행

```bash
# 프로젝트 루트에서
npm run dev:api
```

서버 정상 기동 확인:

```
http://localhost:4000/api/v1/health
# 응답: { "data": { "status": "ok" } }
```

Android 에뮬레이터에서 API 접근 주소: `http://10.0.2.2:4000/api/v1`  
실기기에서 API 접근 주소: 로컬 IP 사용 (예: `http://192.168.x.x:4000/api/v1`)

#### 2. 앱 실행

```bash
cd apps/mobile-app
flutter pub get
flutter run -d <device_id>
```

디바이스 목록 확인:

```bash
flutter devices
```

#### 3. 테스트 계정 및 데이터

| 항목 | 값 |
|---|---|
| 테스트 회원 번호 | `01011110002` |
| 테스트 운영진 번호 | `01011110001` |
| OTP (개발 환경) | `123456` |
| 테스트 모임명 | 토요 풋살 |
| `clubId` | 로컬 JSON에서 확인 (`apps/api/data/clubs.json`) |
| `memberId` | 로컬 JSON에서 확인 (`apps/api/data/members.json`) |

> 테스트 데이터가 없을 경우, 관리자 웹(`http://localhost:3000`)에서 운영진 계정으로 회원 등록 후 진행한다.

### 화면별 수동 테스트

#### T-01. 휴대폰 인증 로그인

**절차**

1. 앱 실행 후 로그인 화면 진입 확인
2. 국가 코드 `+82` 선택
3. 테스트 번호 `01011110002` 입력
4. OTP 요청 버튼 탭
5. OTP 입력 필드에 `123456` 입력
6. 확인 버튼 탭

**기대 결과**

- 홈 화면 또는 프로필 입력 화면으로 이동
- 기존 회원: 가입된 모임이 1개이면 홈으로 바로 진입
- 신규 회원: 이름 등 프로필 입력 화면 표시

**실패 시 확인할 것**

- API 로그에서 `POST /api/v1/auth/otp/request`, `POST /api/v1/auth/otp/verify` 요청 확인
- 에뮬레이터인 경우 baseUrl이 `http://10.0.2.2:4000`인지 확인 (`lib/core/api_client.dart` 등)
- OTP 개발 모드 활성화 여부 확인 (`apps/api/src/mvp.repository.ts`)

---

#### T-02. 홈 대시보드 확인

**절차**

1. 로그인 완료 후 홈 탭 진입
2. 다음 일정 카드 표시 확인
3. 미납 회비 카드 표시 확인
4. 미확인 공지 카드 표시 확인
5. 각 카드 탭 시 해당 화면으로 이동 확인

**기대 결과**

- 세 카드 모두 데이터가 표시됨 (또는 "없음" 표시)
- 운영진 전용 공지/일정은 일반 회원 홈에 미표시

**실패 시 확인할 것**

- API 로그에서 `GET /api/v1/clubs/{clubId}/member-app/{memberId}` 응답 확인
- `clubId`, `memberId`가 올바른 값인지 확인
- 응답 JSON 필드 구조 확인

---

#### T-03. 여러 모임 가입 상태에서 모임 전환

> 테스트 계정이 두 개 이상의 모임에 가입된 경우에만 실행 가능하다.

**절차**

1. 홈 화면 상단 모임명 탭
2. 모임 목록 드롭다운/시트 확인
3. 다른 모임 선택
4. 전환 후 홈 데이터가 선택한 모임 기준으로 갱신되는지 확인
5. 역할이 다른 경우 메뉴 구성 변경 확인

**기대 결과**

- 전환 즉시 모든 탭 데이터가 선택한 모임 기준으로 갱신됨
- 다음 앱 실행 시 마지막 선택 모임으로 자동 진입

**실패 시 확인할 것**

- 로컬 스토리지에 선택 모임 저장 여부 확인
- 전환 후 이전 모임 데이터가 잔류하지 않는지 확인

---

#### T-04. 일정 참석/불참 응답

**사전 조건**: 다가오는 일정이 1건 이상 존재 (관리자 웹에서 생성)

**절차**

1. 하단 탭 > 일정 탭 진입
2. 미응답 일정 확인 (뱃지 표시)
3. 일정 탭하여 상세 진입
4. 참석 버튼 탭 → 상태 변경 확인
5. 불참으로 변경 → 상태 변경 확인
6. 응답 마감일이 지난 일정에서 버튼 비활성화 확인

**기대 결과**

- 응답이 즉시 저장되고 버튼 상태가 변경됨
- 마감일 초과 시 버튼 비활성화, 안내 문구 표시

**실패 시 확인할 것**

- API 로그에서 `PATCH /api/v1/clubs/{clubId}/events/{eventId}/responses` 확인
- 응답 본문에 `memberId`, `response` 필드 포함 여부 확인

---

#### T-05. 공지 확인 처리

**사전 조건**: 운영진이 공지를 1건 이상 작성한 상태

**절차**

1. 하단 탭 > 공지 탭 진입
2. 미확인 공지 뱃지 확인
3. 공지 항목 탭하여 상세 진입
4. 내용 읽기 (자동 확인 처리)
5. 목록으로 돌아와 뱃지 해제 확인
6. 관리자 웹에서 해당 공지 확인률이 증가했는지 확인

**기대 결과**

- 상세 진입 시 즉시 읽음 처리됨
- 공지 목록에서 해당 공지 뱃지 사라짐

**실패 시 확인할 것**

- API 로그에서 `PATCH /api/v1/clubs/{clubId}/notices/{noticeId}/read` 확인
- 응답 본문에 `memberId` 포함 여부 확인

---

#### T-06. 공지 좋아요·댓글

**사전 조건**: 공지 상세 화면 진입 상태

**절차**

1. 공지 상세 하단 좋아요 버튼 탭
2. 좋아요 수 증가 확인
3. 다시 탭하여 좋아요 해제
4. 댓글 입력 필드 탭
5. 댓글 내용 입력 후 등록
6. 댓글 목록에서 새 댓글 확인
7. 본인 댓글 삭제 버튼 표시 확인

**기대 결과**

- 좋아요 토글 즉시 반영
- 댓글이 목록 최하단에 추가됨

**실패 시 확인할 것**

- `PATCH .../reactions`, `POST .../comments` API 응답 확인

---

#### T-07. 회비 납부 상태 확인

**사전 조건**: 운영진이 회비 항목을 생성하고 납부 상태가 존재하는 상태

**절차**

1. 하단 탭 > 회비 탭 진입
2. 회비 목록 확인 (항목명, 금액, 납부 기한, 상태)
3. 미납 항목 표시 방식 확인 (색상 구분 등)
4. 납부 완료 항목 확인
5. 다른 회원의 회비 정보가 노출되지 않는지 확인

**기대 결과**

- 본인에게 청구된 항목만 표시됨
- 납부 상태 구분이 명확함 (미납/납부/면제)

**실패 시 확인할 것**

- `GET /api/v1/clubs/{clubId}/member-app/{memberId}` 응답 내 회비 필드 확인
- 타 회원 회비 데이터 미포함 여부 확인

---

#### T-08. 프로필 수정 저장

**절차**

1. 하단 탭 > 더보기 > 프로필 수정 진입
2. 이름 수정
3. 생년월일 수정 (지원 시)
4. 저장 버튼 탭
5. 홈 화면으로 돌아와 이름 변경 반영 확인

**기대 결과**

- 저장 후 앱 전체에서 변경된 이름 표시

**실패 시 확인할 것**

- `PATCH /api/v1/members/{memberId}/profile` 요청 확인
- 응답에 업데이트된 프로필 데이터 포함 여부 확인

---

#### T-09. 공개 모임 가입 신청

**사전 조건**: 가입되지 않은 공개 모임이 존재하거나 공개 모임 링크 보유

**절차**

1. 더보기 탭 > 모임 가입 진입 (또는 공개 모임 링크 접근)
2. 모임 정보 확인
3. 이름, 연락처 입력 (기존 프로필 자동 채움 확인)
4. 가입 인사 입력 (선택)
5. 신청 제출
6. "신청이 완료되었습니다" 안내 확인
7. 관리자 웹에서 가입 신청 대기 목록 확인

**기대 결과**

- 신청 접수 완료 안내 표시
- 관리자 웹 가입 신청 목록에 `pending` 상태로 등록됨

**실패 시 확인할 것**

- `POST /api/v1/clubs/{clubId}/join-requests` 요청/응답 확인
- 이미 가입/신청 중인 경우 중복 방지 처리 확인

---

#### T-10. 초대 코드 가입

**사전 조건**: 관리자 웹에서 유효한 초대 링크 생성 완료

**절차**

1. 초대 링크 URL 탭 (딥링크) 또는 더보기 탭 > 초대 코드 입력
2. 모임 정보 확인 화면 표시 확인
3. 가입 버튼 탭
4. 가입 완료 후 내 모임 목록에 추가 확인
5. 만료된 링크로 재시도 → 오류 안내 확인
6. 비활성화된 링크로 재시도 → 오류 안내 확인

**기대 결과**

- 유효한 링크: 가입 완료 후 해당 모임 홈 화면으로 이동
- 만료 링크: "초대 링크가 만료되었습니다" 오류 표시
- 비활성화 링크: "유효하지 않은 초대 링크입니다" 오류 표시

**실패 시 확인할 것**

- `POST /api/v1/clubs/{clubId}/invite-links/{token}/accept` 요청 확인
- 토큰 검증 로직에서 만료/비활성화 조건 처리 확인

```
├── clubs.json        # 모임 목록
├── members.json      # 회원 목록

├── events.json       # 일정 목록
└── notices.json      # 공지 목록
```

#### 자주 발생하는 문제

| 증상 | 원인 | 해결 방법 |
|---|---|---|
| 앱에서 API 연결 실패 | baseUrl 설정 오류 | 에뮬레이터는 `10.0.2.2`, 실기기는 로컬 IP 사용 |
| OTP 입력 후 오류 | 개발 OTP 미작동 | API가 `123456` 허용 모드인지 확인 |
| 홈 화면 데이터 없음 | `clubId`/`memberId` 불일치 | JSON 파일에서 실제 ID 값 확인 |
| 공지 확인 뱃지 미해제 | read API 미호출 | 상세 화면 진입 시 자동 read 처리 로직 확인 |
| 초대 링크 오류 | 링크 만료/비활성화 | 관리자 웹에서 새 링크 생성 후 재시도 |
