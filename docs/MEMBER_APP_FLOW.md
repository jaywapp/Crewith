# Crewith 회원 앱 사용 흐름

## 작성 기준

- 구현 전 설계 기준으로 작성한다.
- 각 플로우는 목적, 사용자, 진입 경로, 실행 단계, 성공 결과, 예외/오류 상황으로 구성한다.
- 앱은 Android 우선, iOS는 Beta 1 이후 검증 예정이다.
- 오프라인 시 로컬 임시 저장 후 동기화 동작을 별도로 명시한다.

---

## 1. 휴대폰 인증 로그인

**목적**: 회원이 휴대폰 번호로 앱에 인증하여 모임 기능에 접근한다.

**사용자**: 신규 가입자 / 기존 회원

**진입 경로**: 앱 최초 실행 또는 로그아웃 후 재진입

**실행 단계**
1. 앱 실행 후 휴대폰 번호 입력 화면 표시
2. 국가 코드 선택 (+82) 및 번호 입력
3. OTP 요청 버튼 탭
4. SMS로 수신한 6자리 OTP 입력 (개발 환경: `123456`)
5. 확인 버튼 탭

**성공 결과**
- **기존 회원**: 가입된 모임이 1개이면 해당 모임 홈 화면으로 이동. 여러 모임이면 모임 선택 화면 표시.
- **신규 회원**: 프로필 입력 화면으로 이동
- JWT 토큰이 앱 로컬 스토리지에 저장됨

**예외/오류 상황**
- 잘못된 OTP: 오류 메시지 표시, 재입력 요청
- OTP 만료 (5분): 재발송 버튼 활성화
- 네트워크 없음: "인터넷 연결을 확인해 주세요" 안내
- 서비스 미등록 번호로 첫 진입: 신규 사용자 프로필 입력 플로우 진행

---

## 2. 가입된 모임 목록 확인

**목적**: 회원이 가입한 모임들을 확인하고 관리한다.

**사용자**: 로그인한 회원

**진입 경로**: 로그인 후 여러 모임 가입 시 자동 진입 / 홈 화면 상단 모임명 탭

**실행 단계**
1. 모임 목록 화면 진입
2. 가입된 모임 카드 목록 확인 (모임명, 종목, 역할, 구독 상태)
3. 특정 모임 카드 탭

**성공 결과**
- 가입된 모임 목록이 역할과 함께 표시됨 (모임장, 운영진, 일반 회원)
- 선택한 모임으로 전환되어 해당 모임 홈 화면으로 이동

**예외/오류 상황**
- 가입된 모임 없음: "가입된 모임이 없습니다" + 모임 가입 방법 안내
- suspended 모임: 목록에 표시되나 진입 불가, 안내 메시지 표시

---

## 3. 모임 전환

**목적**: 여러 모임에 가입한 회원이 현재 활성 모임을 변경한다.

**사용자**: 두 개 이상 모임에 가입한 회원

**진입 경로**: 홈 화면 상단 모임명 탭 > 모임 전환 드롭다운 또는 모임 목록 화면

**실행 단계**
1. 상단 모임명 또는 전환 아이콘 탭
2. 가입된 모임 목록 드롭다운/시트 표시
3. 전환할 모임 선택

**성공 결과**
- 선택한 모임으로 즉시 전환
- 전체 탭 데이터가 선택한 모임 기준으로 갱신됨
- 마지막 선택 모임이 앱 로컬에 저장되어 다음 실행 시에도 유지

**예외/오류 상황**
- 모임 전환 중 네트워크 없음: 캐시된 데이터로 화면 표시, 동기화 대기 표시

---

## 4. 홈 대시보드 확인

**목적**: 회원이 현재 모임의 주요 현황을 한 화면에서 파악한다.

**사용자**: 로그인한 회원 (모든 역할)

**진입 경로**: 앱 하단 탭 > 홈 탭

**실행 단계**
1. 홈 탭 진입
2. 모임명, 내 역할 확인
3. 다음 일정 카드 확인
4. 미납 회비 건수 확인
5. 미확인 공지 건수 확인
6. 각 카드 탭 시 해당 화면으로 이동

**성공 결과**
- 다음 일정 정보가 날짜, 제목, 장소와 함께 표시됨 (일정 없으면 없음 표시)
- 미납 회비 건수가 표시됨 (0이면 "모두 납부 완료" 표시)
- 미확인 공지 건수가 표시됨

**예외/오류 상황**
- API 응답 오류: 캐시된 마지막 데이터 표시 + 갱신 실패 안내
- 운영진 전용 공지/일정은 일반 회원 홈에 미표시

---

## 5. 일정 참석/불참 응답

**목적**: 회원이 다가오는 일정에 대한 참석 여부를 사전에 알린다.

**사용자**: 로그인한 회원 (모든 역할)

**진입 경로**: 하단 탭 > 일정 탭 > 일정 선택 또는 홈 > 다음 일정 카드 탭

**실행 단계**
1. 일정 목록에서 응답 미완료 일정 확인 (뱃지 표시)
2. 일정 상세 화면 진입
3. 참석 또는 불참 버튼 탭
4. 응답 완료 후 상태 변경 확인
5. 불참 시 이유 입력 (선택 사항)

**성공 결과**
- 응답이 저장되고 버튼 상태가 변경됨
- 응답 마감일까지 변경 가능
- 오프라인 시 로컬 저장 후 네트워크 복귀 시 동기화

**예외/오류 상황**
- 응답 마감 후 변경 시도: 버튼 비활성화, "응답 마감이 지났습니다" 표시
- 운영진 전용 일정은 일반 회원에게 미표시
- 이미 응답한 경우: 현재 상태 표시 + 변경 버튼 제공 (마감 전)

---

## 6. 실제 출석 상태 확인

**목적**: 회원이 자신의 출석 기록을 확인한다.

**사용자**: 로그인한 회원 (모든 역할)

**진입 경로**: 일정 상세 화면 > 내 출석 탭 또는 일정 목록에서 지난 일정 선택

**실행 단계**
1. 지난 일정 탭 (과거 일정 목록)
2. 특정 일정 선택
3. 내 출석 상태 확인 (출석/지각/결석)
4. 동반 인원 수 확인

**성공 결과**
- 출석 상태가 표시됨 (운영진이 체크한 기록)
- 미체크 시 "미기록" 표시

**예외/오류 상황**
- 출석 체크 미완료 일정: "출석 체크가 아직 완료되지 않았습니다" 표시
- 다른 회원의 출석 상태는 본인에게 미표시 (자신 것만)

---

## 7. 공지 확인

**목적**: 회원이 모임 공지를 확인하고 자동으로 읽음 처리한다.

**사용자**: 로그인한 회원 (모든 역할, 공개 범위 기준)

**진입 경로**: 하단 탭 > 공지 탭 또는 푸시 알림 탭

**실행 단계**
1. 공지 목록 진입 (미확인 공지 뱃지 표시)
2. 공지 항목 탭하여 상세 진입
3. 공지 내용 읽기
4. 상세 화면에서 자동 확인 처리됨
5. 목록으로 돌아와 뱃지 해제 확인

**성공 결과**
- 상세 화면 진입 시 자동으로 읽음 처리됨
- 목록에서 해당 공지의 미확인 뱃지가 사라짐
- 운영진 전용 공지는 일반 회원 목록에 미표시

**예외/오류 상황**
- 오프라인 시 확인 처리: 로컬 상태만 변경, 온라인 복귀 시 서버 동기화
- 삭제된 공지 링크로 진입: "삭제된 공지입니다" 안내

---

## 8. 공지 좋아요/댓글

**목적**: 회원이 공지에 반응하고 의견을 남긴다.

**사용자**: 로그인한 회원 (공지 공개 대상 기준)

**진입 경로**: 공지 상세 화면 > 하단 반응 영역

**실행 단계**
1. 공지 상세 화면 하단에서 좋아요 버튼 탭
2. 좋아요 수 증가 확인 후 다시 탭하여 해제
3. 댓글 입력 필드 탭
4. 댓글 내용 입력 후 등록
5. 댓글 목록에서 새 댓글 확인

**성공 결과**
- 좋아요가 토글됨 (활성화/비활성화)
- 댓글이 목록 최하단에 추가됨
- 작성자 이름과 시각이 표시됨
- 본인 댓글은 삭제 버튼 표시

**예외/오류 상황**
- 댓글 내용 비어 있음: 등록 버튼 비활성화
- 오프라인 시 좋아요/댓글: 로컬 반영 후 동기화 또는 오류 표시 (정책 결정 필요)

---

## 9. 회비 납부 상태 확인

**목적**: 회원이 자신의 회비 납부 현황을 확인한다.

**사용자**: 로그인한 회원 (모든 역할, 본인 데이터만)

**진입 경로**: 하단 탭 > 회비 탭 또는 홈 대시보드 > 미납 회비 카드

**실행 단계**
1. 회비 탭 진입
2. 회비 목록 확인 (항목명, 금액, 납부 기한, 납부 상태)
3. 미납 항목 확인 (빨간색 또는 별도 표시)
4. 납부 완료 항목 확인
5. 면제 항목 확인

**성공 결과**
- 본인에게 청구된 회비 항목만 표시됨
- 납부 상태(미납/납부/면제)가 항목별로 구분됨
- 미납 항목에 납부 기한 경과 여부 표시

**예외/오류 상황**
- 청구된 회비 없음: "청구된 회비가 없습니다" 표시
- 타 회원 회비 정보는 어떤 경로로도 미노출

---

## 10. 프로필 수정

**목적**: 회원이 자신의 앱 프로필 정보를 수정한다.

**사용자**: 로그인한 회원 (모든 역할)

**진입 경로**: 하단 탭 > 더보기 탭 > 프로필 수정

**실행 단계**
1. 프로필 수정 화면 진입
2. 수정 가능한 필드 확인 (이름, 생년월일, 성별, 프로필 사진)
3. 원하는 필드 수정
4. 저장 버튼 탭
5. 홈 화면으로 돌아와 변경 내용 반영 확인

**성공 결과**
- 수정한 필드가 앱 전체에 반영됨
- 프로필 사진 변경 시 업로드 후 URL 저장됨 (Storage 연동 시)

**예외/오류 상황**
- 이름 비어 있음: 저장 불가 안내
- 이미지 크기 초과 (5MB 이상): 업로드 불가 안내 (추후 구현)
- 모임별 공개 설정에 따라 일부 필드가 다른 회원에게 노출되지 않을 수 있음 (확인 문구 표시)

---

## 11. 공개 모임 가입 신청

**목적**: 공개 모임을 발견한 사용자가 가입을 신청한다.

**사용자**: 로그인한 회원 또는 비회원 (앱 내 공개 모임 탐색 추후 구현)

**진입 경로**: 더보기 탭 > 모임 가입 > 공개 모임 신청 또는 공개 모임 링크 공유로 진입

**실행 단계**
1. 공개 모임 가입 화면 진입
2. 모임 정보 확인
3. 신청자 이름, 연락처 입력 (기존 프로필 자동 채움)
4. 가입 인사 입력 (선택)
5. 신청 제출

**성공 결과**
- 신청이 `pending` 상태로 접수됨
- "신청이 완료되었습니다. 운영진 승인 후 가입됩니다." 안내
- 승인 시 FCM 알림 수신 (연동 시)

**예외/오류 상황**
- 이미 가입된 모임 재신청: "이미 가입된 모임입니다" 안내
- 이미 신청 중인 모임 재신청: "이미 신청 중입니다" 안내
- 비공개 모임 URL 직접 접근: 초대 코드 입력 화면으로 안내

---

## 12. 비공개 모임 초대코드 가입

**목적**: 초대 링크를 받은 사용자가 비공개 모임에 가입한다.

**사용자**: 로그인한 회원 또는 신규 사용자

**진입 경로**: 초대 링크 URL 탭 (딥링크) 또는 더보기 탭 > 초대 코드 입력

**실행 단계**
1. 초대 링크 URL 탭 또는 초대 코드 직접 입력
2. 가입 확인 화면에서 모임 이름, 종목 확인
3. 비로그인 상태: 로그인/가입 플로우 진행 후 자동 복귀
4. 가입 버튼 탭
5. 가입 완료 확인

**성공 결과**
- 가입 완료 후 해당 모임이 내 모임 목록에 추가됨
- 해당 모임 홈 화면으로 이동

**예외/오류 상황**
- 만료된 링크: "초대 링크가 만료되었습니다" 안내
- 비활성화된 링크: "유효하지 않은 초대 링크입니다" 안내
- 이미 가입된 모임: "이미 가입된 모임입니다" 안내 후 해당 모임으로 이동
- 비로그인 상태: 로그인 화면으로 이동 후 가입 플로우 재개
