# Crewith MVP 릴리즈 체크리스트

## 작성 기준

- 실제 스포츠 동호회 1개 이상이 사용 가능한 수준을 기준으로 한다.
- 각 항목은 확인 방법과 통과 기준을 포함한다.
- 체크하지 않은 항목이 있으면 릴리즈를 진행하지 않는다.

---

## 1. 로컬 개발 검증

| 항목 | 확인 방법 | 통과 기준 |
|---|---|---|
| API 서버 정상 기동 | `npm run dev:api` 실행 후 `GET /api/v1/health` | HTTP 200, `{ "status": "ok" }` |
| 관리자 웹 정상 기동 | `npm run dev:admin` 실행 후 `http://localhost:3000` | HTTP 200, 화면 렌더링 |
| Flutter 앱 빌드 | `flutter run -d <device>` | 오류 없이 홈 화면 표시 |
| 의존 패키지 설치 | `npm ci` 및 `flutter pub get` | 오류 없이 완료 |
| TypeScript 타입 검사 | `npm run typecheck` | 타입 오류 없음 |
| Prisma 스키마 유효성 | `npx prisma validate` | 오류 없음 |

## 13. 롤백 기준

아래 중 하나라도 해당하면 즉시 롤백한다.

| 조건 | 판단 기준 |
|---|---|
| API 헬스 체크 실패 | 배포 후 5분 내 `/health` HTTP 200 미응답 |
| 인증 불가 | OTP 로그인 5회 연속 실패 |
| 데이터 노출 | 일반 회원에게 다른 회원의 개인정보 또는 운영 데이터 노출 확인 |
| 관리자 웹 빌드 오류 | 주요 라우트 HTTP 500 |
| 앱 크래시 | 홈 화면 진입 전 반복 크래시 |

### 롤백 절차

1. Railway/Vercel 이전 배포 버전으로 즉시 롤백
2. 이슈 문서화 (현상, 재현 조건, 영향 범위)
3. 핫픽스 브랜치에서 수정 후 재배포 게이트 통과 후 재배포
