# Crewith 개발 태스크 분해

## 1. 목적

이 문서는 Crewith MVP 개발을 실제 작업 단위로 분해한다. 각 작업은 관리자 웹, 모바일 앱, 백엔드, DB, 인프라, QA가 병렬로 진행할 수 있도록 구성한다.

## 2. 작업 원칙

- P0 기능부터 구현한다.
- 데이터 모델과 API 계약을 먼저 고정한다.
- 관리자 웹과 모바일 앱은 같은 API 계약을 사용한다.
- 운영진이 반복적으로 쓰는 작업은 웹에서 먼저 완성하고 앱 운영 모드로 확장한다.
- 실제 결제는 MVP에서 제외한다.
