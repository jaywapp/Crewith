# Crewith 사용자 흐름

## 1. 목적

이 문서는 Crewith MVP에서 반드시 검증해야 하는 주요 사용자 흐름을 정의한다. 각 흐름은 화면 설계, API 구현, QA 시나리오의 기준으로 사용한다.

## 2. 사용자 유형

- 모임장: 모임 생성, 전체 설정, 운영진 권한 관리
- 운영진: 구성원, 회비, 일정, 공지, 출석 관리
- 일반회원: 일정, 회비, 공지, 구성원 정보 확인과 응답
- 게스트/신규 사용자: 가입 전 인증, 초대 링크 또는 가입 신청

## 3. 공통 흐름

### 3.1 휴대폰 인증 로그인

1. 사용자가 휴대폰 번호를 입력한다.
2. 시스템이 SMS 인증번호를 발송한다.
3. 사용자가 인증번호를 입력한다.
4. 시스템이 인증번호를 검증한다.
5. 기존 사용자면 토큰을 발급하고 홈으로 이동한다.
6. 신규 사용자면 프로필 입력으로 이동한다.

예외:

- 인증번호 만료
- 인증번호 불일치
- SMS 발송 실패
- 과도한 재시도

### 3.2 프로필 입력

1. 신규 사용자가 이름, 생년월일, 성별을 입력한다.
2. 선택적으로 프로필 사진을 등록한다.
3. 시스템이 프로필을 저장한다.
4. 모임 선택 또는 모임 생성 화면으로 이동한다.

## 4. 모임 흐름

### 4.1 모임 생성

1. 사용자가 모임 생성을 선택한다.
2. 모임명, 스포츠 종목, 공개/비공개 여부를 입력한다.
3. 시스템이 모임을 생성한다.
4. 생성자는 `owner`가 된다.
5. 30일 무료 체험이 시작된다.
6. 대시보드로 이동한다.

성공 기준:

- 모임이 생성된다.
- 생성자가 모임장 권한을 가진다.
- 구독 상태가 trial로 표시된다.

### 4.2 모임 전환

1. 사용자가 현재 모임명을 선택한다.
2. 가입한 모임 목록을 확인한다.
3. 다른 모임을 선택한다.
4. 앱/웹의 모든 데이터 컨텍스트가 선택한 모임으로 변경된다.

주의:

- 사용자는 모임마다 다른 권한을 가질 수 있다.
- 모임 전환 시 권한 기반 메뉴가 갱신되어야 한다.

### 4.3 공개 모임 가입 신청

1. 사용자가 공개 모임 가입 링크로 진입한다.
2. 로그인하지 않았다면 휴대폰 인증을 진행한다.
3. 간단한 인적사항과 가입 인사를 입력한다.
4. 가입 신청을 제출한다.
5. 운영진은 가입 신청 목록에서 내용을 확인한다.
6. 운영진이 승인 또는 거절한다.
7. 시스템이 사용자에게 앱 푸시 알림을 보낸다.
8. 승인된 사용자는 모임에 일반회원으로 가입된다.

예외:

- 이미 가입한 사용자
- 이미 대기 중인 신청
- 모임이 비공개로 변경됨

### 4.4 비공개 초대 링크 가입

1. 운영진이 만료 기간을 설정해 초대 링크를 생성한다.
2. 초대 대상자가 링크로 진입한다.
3. 로그인하지 않았다면 휴대폰 인증을 진행한다.
4. 시스템이 링크 유효성을 검증한다.
5. 사용자가 가입을 확정한다.
6. 시스템이 일반회원으로 등록한다.

예외:

- 링크 만료
- 링크 비활성화
- 이미 가입한 사용자

## 5. 구성원 흐름

### 5.1 Excel 명부 업로드

1. 운영진이 회원 목록에서 Excel 업로드를 선택한다.
2. 파일을 선택한다.
3. 시스템이 필수값, 연락처 중복, 형식을 검증한다.
4. 정상 행은 즉시 등록한다.
5. 실패 행은 오류 목록으로 표시한다.

성공 기준:

- 정상 행은 회원으로 등록된다.
- 실패 행은 행 번호와 사유가 표시된다.

### 5.2 회원 상태 변경

1. 운영진이 회원 상세를 연다.
2. 상태를 활동 중, 휴면, 탈퇴, 강퇴 중 하나로 변경한다.
3. 탈퇴/강퇴/휴면이면 개인정보 삭제 예정일을 30일 후로 설정한다.
4. 회비/참석 이력은 보존한다.

## 6. 회비 흐름

### 6.1 월회비 설정

1. 운영진이 월회비 설정을 연다.
2. 금액, 납부일, 납부 간격, 미납 유예기간을 입력한다.
3. 미납 알림 자동 발송 여부와 발송 시점을 설정한다.
4. 시스템이 설정을 저장한다.
5. 백그라운드 작업이 납부일/간격 기준으로 회비 항목을 생성한다.

### 6.2 일회성 비용 생성

1. 운영진이 일회성 비용 생성을 선택한다.
2. 항목명, 금액, 납부일을 입력한다.
3. 부과 대상을 전체 회원 또는 특정 회원으로 선택한다.
4. 특정 회원이면 운영진이 수동으로 대상자를 선택한다.
5. 시스템이 대상자별 납부 상태를 생성한다.

제외:

- 일정 참석자 자동 부과

### 6.3 회비 납부 토글

1. 운영진이 회비 현황을 연다.
2. 회비 항목을 선택한다.
3. 회원별 납부 상태를 토글한다.
4. 납부로 변경하면 오늘 날짜가 자동 기록된다.
5. 운영진은 필요하면 납부일을 수정한다.
6. 대시보드와 통계가 갱신된다.

### 6.4 미납 알림

1. 시스템이 납부일과 유예기간을 기준으로 미납자를 계산한다.
2. 모임별 알림 설정을 확인한다.
3. 자동 발송 조건에 맞는 대상에게 앱 푸시를 보낸다.
4. 사용자는 알림을 통해 내 회비 화면으로 이동한다.

## 7. 일정 흐름

### 7.1 일정 생성

1. 운영진이 일정 생성을 선택한다.
2. 제목, 설명, 시작/종료 일시를 입력한다.
3. 네이버 지도에서 장소를 검색하고 선택한다.
4. 참석 응답 마감을 설정한다.
5. 보기 권한을 전체 회원 또는 운영진만으로 설정한다.
6. 일정을 저장한다.
7. 대상 회원에게 앱 푸시를 보낸다.

### 7.2 참석 의사 응답

1. 회원이 일정 상세를 연다.
2. 참석 또는 불참을 선택한다.
3. 응답 마감 전까지 수정할 수 있다.
4. 마감 이후에는 수정할 수 없다.

### 7.3 출석부 체크

1. 운영진이 일정 상세에서 출석부를 연다.
2. 사전 참석 의사 현황을 확인한다.
3. 회원별 실제 상태를 참석, 지각, 결석으로 체크한다.
4. 동반 인원을 입력한다.
5. 저장한다.
6. 실제 출석률과 참석 의사 대비 출석률이 갱신된다.

## 8. 공지 흐름

### 8.1 공지 작성

1. 운영진이 공지 작성을 선택한다.
2. 제목과 본문을 입력한다.
3. 보기 권한을 전체 회원 또는 운영진만으로 설정한다.
4. 공지를 게시한다.
5. 대상 회원에게 앱 푸시를 보낸다.

주의:

- 운영진 전용 공지는 일반 회원 목록에 노출하지 않는다.

### 8.2 공지 확인

1. 회원이 공지 목록을 연다.
2. 공지 상세에 진입한다.
3. 시스템이 자동으로 확인 처리한다.
4. 운영진은 확인/미확인 목록과 확인률을 확인한다.

### 8.3 댓글과 좋아요

1. 보기 권한이 있는 회원이 공지 상세에 진입한다.
2. 댓글을 작성한다.
3. 좋아요를 누른다.
4. 운영진과 작성자는 댓글을 관리할 수 있다.

### 8.4 미확인 리마인드

1. 시스템이 공지별 미확인 회원을 계산한다.
2. 모임별 리마인드 설정을 확인한다.
3. 조건에 맞으면 미확인 회원에게 앱 푸시를 보낸다.

## 9. 운영진 앱 모드 흐름

### 9.1 모바일 회비 체크

1. 운영진이 앱 홈에서 운영 모드로 진입한다.
2. 회비 체크를 선택한다.
3. 회비 항목을 선택한다.
4. 회원별 납부 상태를 토글한다.
5. 저장 결과를 확인한다.

### 9.2 모바일 출석부

1. 운영진이 오늘 일정에서 출석부를 연다.
2. 회원별 실제 출석 상태를 체크한다.
3. 동반 인원을 입력한다.
4. 저장한다.

## 10. QA 기준 흐름

MVP QA는 최소 아래 흐름을 end-to-end로 검증해야 한다.

- 신규 사용자 휴대폰 인증 후 모임 생성
- 공개 모임 가입 신청 후 승인
- 비공개 초대 링크 가입
- Excel 명부 업로드
- 월회비 설정 후 납부 상태 생성
- 일회성 비용 생성 후 납부 토글
- 일정 생성 후 참석 응답과 출석부 저장
- 공지 작성 후 확인 여부와 댓글/좋아요
- 미납/일정/공지 알림 발송
- 운영진 전용 공지/일정 일반 회원 숨김
