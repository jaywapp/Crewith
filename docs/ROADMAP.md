# Crewith 로드맵

## 1. 로드맵 기준

Crewith는 10~30명 규모의 스포츠 동호회를 첫 타깃으로 한다. MVP는 운영진이 Excel, 카카오톡 캘린더, 채팅방, 수기 회비 장부로 나누어 관리하던 일을 하나의 서비스에서 처리할 수 있는지 검증한다.

## 2. 단계 요약

| 단계 | 목표 | 결과물 |
|---|---|---|
| Phase 0 | 개발 기반 구축 | 모노레포, 배포 환경, DB, 인증 골격 |
| Phase 1 | 기본 사용 가능 | 로그인, 프로필, 모임 생성/전환 |
| Phase 2 | 운영진 명부 관리 | 구성원, 가입 신청, 초대 링크, Excel 업로드 |
| Phase 3 | 회비 관리 | 월회비, 일회성 비용, 납부 토글, 미납 관리 |
| Phase 4 | 일정/출석 관리 | 일정, 참석 의사, 출석부, 지도 |
| Phase 5 | 공지/알림 | 공지, 댓글, 좋아요, 확인 여부, FCM |
| Phase 6 | 대시보드 | 통계, 구독 상태, 운영 작업 큐 |
| Phase 7 | MVP 안정화 | QA, 권한 검증, Android 테스트, 배포 |

## 3. MVP 출시 범위

MVP에 포함한다.

- 휴대폰 인증
- 모임 생성
- 다중 모임 가입/전환
- 공개 모임 가입 신청
- 비공개 초대 링크
- 구성원 명부 관리
- Excel 업로드
- 모임별 권한
- 월회비/일회성 비용 관리
- 납부 상태 자동 생성
- 납부 토글
- 미납 알림
- 일정 생성
- 네이버 지도 연동
- 참석/불참 응답
- 운영진 출석부
- 공지 작성
- 댓글/좋아요
- 공지 확인 여부
- 공지 미확인 리마인드
- 앱 푸시
- 운영 대시보드
- 회원 앱
- 관리자 웹

MVP에서 제외한다.

- 실제 결제
- 서비스 내 회비 결제
- 회원 직접 체크인
- 공개 모임 검색/탐색
- 일정 참석자 대상 자동 회비 부과
- 복잡한 커스텀 권한
- 실시간 채팅
- 고급 리포트
- iOS 정식 최적화

## 4. Phase 0. 기반 구축

목표:

- 개발을 시작할 수 있는 저장소, 배포, DB 기반을 만든다.

작업:

- 모노레포 구성
- Next.js 관리자 웹 생성
- Flutter 앱 생성
- NestJS API 생성
- PostgreSQL 연결
- Prisma 마이그레이션 설정
- Railway/Vercel 배포 연결
- FCM 프로젝트 생성
- R2/S3 버킷 생성

완료 기준:

- 관리자 웹이 배포된다.
- API health check가 배포 환경에서 응답한다.
- DB migration이 적용된다.
- Flutter 앱이 개발 기기에서 실행된다.

## 5. Phase 1. 계정과 모임

목표:

- 사용자가 로그인하고 모임을 만들 수 있다.

작업:

- SMS 인증
- JWT 인증
- 프로필 입력
- 모임 생성
- 모임 목록
- 모임 전환
- 무료 체험 상태 표시

완료 기준:

- 신규 사용자가 휴대폰 인증 후 모임을 생성한다.
- 사용자가 여러 모임을 전환할 수 있다.
- 생성자는 모임장 권한을 가진다.

## 6. Phase 2. 구성원과 가입

목표:

- 운영진이 구성원을 관리하고 기존 명부를 가져올 수 있다.

작업:

- 회원 목록
- 회원 상세
- 상태 변경
- 역할 변경
- Excel 업로드
- 공개 가입 신청
- 승인/거절
- 비공개 초대 링크
- 개인정보 공개 설정

완료 기준:

- 운영진이 Excel 명부를 업로드한다.
- 공개 가입 신청을 승인/거절한다.
- 초대 링크로 비공개 모임에 가입한다.
- 일반 회원은 설정된 공개 범위만 볼 수 있다.

## 7. Phase 3. 회비

목표:

- 운영진이 회비를 설정하고 미납자를 관리할 수 있다.

작업:

- 월회비 설정
- 납부일/간격/유예기간
- 납부 상태 자동 생성
- 일회성 비용
- 특정 회원 수동 대상 지정
- 납부 토글
- 납부일 수정
- 미납자 목록
- 미납 알림

완료 기준:

- 회비 항목과 회원별 납부 상태가 생성된다.
- 운영진이 납부 상태를 토글한다.
- 미납자가 계산된다.
- 회원이 본인의 회비 상태를 확인한다.

## 8. Phase 4. 일정과 출석

목표:

- 일정 공지, 참석 의사, 실제 출석부가 연결된다.

작업:

- 일정 CRUD
- 네이버 지도 장소 검색
- 일정 보기 권한
- 참석/불참 응답
- 응답 마감
- 출석부
- 동반 인원
- 일정 알림

완료 기준:

- 회원이 참석/불참을 응답한다.
- 운영진이 출석부를 체크한다.
- 실제 출석률과 참석 의사 대비 출석률이 계산된다.

## 9. Phase 5. 공지와 알림

목표:

- 공지 전달과 확인 여부 추적을 제공한다.

작업:

- 공지 CRUD
- 보기 권한
- 댓글
- 좋아요
- 열람 시 자동 확인
- 확인/미확인 목록
- 미확인 리마인드
- FCM 발송
- 앱 알림 목록

완료 기준:

- 회원이 공지를 확인하면 자동으로 확인 처리된다.
- 운영진이 공지 확인률과 미확인 회원을 본다.
- 미확인 리마인드가 발송된다.

## 10. Phase 6. 통계와 운영 대시보드

목표:

- 운영진이 모임 운영 상태를 한 화면에서 파악한다.

작업:

- 전체 회원 수
- 활성 회원 수
- 회비 미납자 수
- 실제 출석률
- 참석 의사 대비 출석률
- 공지 확인률
- 월별 회비 수납률
- 작업 큐
- 구독/무료 체험 상태

완료 기준:

- 대시보드에서 주요 운영 지표가 보인다.
- 가입 신청, 미납자, 일정, 공지 리마인드 작업을 바로 찾을 수 있다.

## 11. Phase 7. MVP 안정화

목표:

- 실제 스포츠 동호회 1개 이상이 사용 가능한 수준으로 안정화한다.

작업:

- 핵심 E2E 테스트
- 권한 테스트
- 알림 테스트
- Android 실제 기기 테스트
- 관리자 웹 반응형 점검
- 오류 메시지 정리
- 로그와 모니터링
- 개인정보 삭제 작업 검증

완료 기준:

- P0 흐름이 모두 통과한다.
- 운영진 전용 데이터가 일반 회원에게 노출되지 않는다.
- 알림과 백그라운드 작업이 동작한다.
- 파일 업로드가 동작한다.

## 12. Beta 이후 확장

### Beta 1

- iOS 검증
- 실제 구독 결제 준비
- SMS/PASS 본인인증 비교
- 회계 담당자/일정 담당자 세부 권한 검토
- 공지 첨부파일

### Beta 2

- 공개 모임 검색/탐색
- 회원 직접 체크인
- 일정 참석자 대상 회비 자동 부과
- 고급 통계
- 운영 리포트 다운로드

### 정식 출시 후보

- 실제 구독 결제
- 결제 실패/구독 만료 정책
- 고객지원/문의 관리
- 약관/개인정보처리방침 정식화
- 운영자 어드민
- 장애 대응 프로세스

## 13. 성공 지표

MVP에서 확인할 지표:

- 모임 생성 수
- 활성 모임 수
- 모임당 평균 회원 수
- Excel 업로드 사용률
- 월회비 설정 완료율
- 회비 납부 상태 업데이트 수
- 일정 참석 응답률
- 실제 출석부 작성률
- 공지 확인률
- 미납 알림 클릭률
- 30일 무료 체험 후 유지율

## 14. 주요 리스크

| 리스크 | 대응 |
|---|---|
| 운영 기능 범위가 커짐 | MVP 제외 범위를 유지 |
| 권한 누락으로 정보 노출 | 권한 테스트를 P0로 관리 |
| 알림 과다 발송 | 모임별 알림 설정 제공 |

| 회비/출석 통계 불일치 | 데이터 생성 규칙과 집계 기준 테스트 |
| 모바일 운영 기능 복잡도 | 웹 우선 구현 후 앱 운영 모드 축소 적용 |
