
# Claude 작업 요청: Crewith 1차 MVP 문서 보강

  Crewith 프로젝트의 1차 MVP 문서 작업을 진행해줘.

  ## 작업 원칙

  - **코드는 수정하지 말고 `docs` 폴더 안의 문서만 작성/보강할 것**
  - `apps`, `packages`, 설정 파일, 빌드 파일은 수정하지 말 것
  - 문서는 **한국어**로 작성할 것
  - 현재 MVP 범위는 **동호회/모임 운영 관리 서비스**임
  - 서비스는 **관리자 웹 + 회원 앱**을 동시에 제공함
  - 초기 타깃은 **국내 스포츠 동호회**
  - 네이버 지도 연동과 실제 결제는 후순위로 둠

  ## 작업 범위

  ### 1. `docs/QA_CHECKLIST.md` 보강

  기존 QA 체크리스트를 아래 형식으로 상세화해줘.

  각 항목마다 다음 내용을 포함:

  - 사전 조건
  - 실행 단계
  - 기대 결과
  - 실패 시 확인할 것

  대상 영역:

  - 관리자 웹
  - 회원 앱
  - API
  - 권한/보안
  - 릴리즈 게이트

  ### 2. `docs/FIREBASE_ARCHITECTURE.md` 보강

  Firebase 전환 설계를 상세화해줘.

  포함할 내용:

  - Firestore 컬렉션 구조 초안
  - 컬렉션별 주요 필드
  - 모임별 데이터 분리 방식
  - 회원별 여러 모임 가입 구조
  - 역할 기반 권한 구조
  - Firebase Auth 사용 방식
  - SMS 인증 MVP 방식
  - FCM 토큰 저장 구조
  - Cloud Functions 책임 범위
  - Firestore Security Rules 초안
  - 로컬 개발/운영 환경 분리 방식

  ### 3. `docs/ADMIN_USER_FLOW.md` 신규 작성

  관리자 웹 사용 흐름 문서를 새로 작성해줘.

  포함할 플로우:

  - 관리자 로그인
  - 대시보드 확인
  - 회원 등록
  - 회원 상태 변경
  - 역할 변경
  - 회비 생성
  - 납부 상태 변경
  - 미납자 확인
  - 일정 생성
  - 참석 의사 확인
  - 실제 출석 체크
  - 공지 작성
  - 공지 확인률 확인
  - 가입 신청 승인/거절
  - 초대 링크 생성
  - 리마인더 발송

  각 플로우는 아래 형식으로 작성:

  - 목적
  - 사용자
  - 진입 경로
  - 실행 단계
  - 성공 결과
  - 예외/오류 상황

  ### 4. `docs/MEMBER_APP_FLOW.md` 신규 작성

  회원 앱 사용 흐름 문서를 새로 작성해줘.

  포함할 플로우:

  - 휴대폰 인증 로그인
  - 가입된 모임 목록 확인
  - 모임 전환
  - 홈 대시보드 확인
  - 일정 참석/불참 응답
  - 실제 출석 상태 확인
  - 공지 확인
  - 공지 좋아요/댓글
  - 회비 납부 상태 확인
  - 프로필 수정
  - 공개 모임 가입 신청
  - 비공개 모임 초대코드 가입

  각 플로우는 아래 형식으로 작성:

  - 목적
  - 사용자
  - 진입 경로
  - 실행 단계
  - 성공 결과
  - 예외/오류 상황

  ### 5. `docs/MVP_RELEASE_CHECKLIST.md` 신규 작성

  MVP 릴리즈 체크리스트를 새로 작성해줘.

  포함할 영역:

  - 로컬 개발 검증
  - API 테스트
  - 관리자 웹 빌드
  - 회원 앱 테스트
  - 수동 QA
  - 환경변수 확인
  - Firebase 설정 확인
  - 보안 규칙 확인
  - 개인정보 처리 확인
  - 권한 검증
  - 배포 전 데이터 초기화
  - 배포 후 스모크 테스트
  - 롤백 기준

  ## 주의 사항

  - 현재 Codex가 API의 `clubId` 기반 다중 모임 구조를 수정 중이므로, `apps/api` 코드는 건드리지 말 것
  - 문서에서 실제 구현 완료 여부를 단정하지 말 것
  - 구현 전 설계, QA 기준, 릴리즈 기준으로 작성할 것
  - “현재 구현됨”, “추후 구현” 같은 표현은 명확히 구분할 것
  - 모호한 내용은 합리적으로 가정하되, “가정” 섹션에 따로 적을 것
