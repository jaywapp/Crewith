# Claude Task 03: 관리자 웹 및 회원 앱 사용자 흐름 문서화

## 목표

Crewith 1차 MVP의 실제 사용 흐름을 관리자 웹과 회원 앱으로 나누어 정리한다. 이후 UI/QA/개발 기준으로 바로 사용할 수 있어야 한다.

## 작업 범위

생성 또는 수정할 파일:

- `docs/ADMIN_USER_FLOW.md`
- `docs/MEMBER_APP_FLOW.md`
- 필요 시 `docs/USER_FLOWS.md`

## 관리자 웹 플로우

`docs/ADMIN_USER_FLOW.md`에 아래 플로우를 작성한다.

- 관리자 로그인
- 대시보드 확인
- 회원 등록
- 회원 상태 변경
- 역할 변경
- 회비 생성
- 납부 상태 변경
- 미납자 확인
- 일정 생성
- 참석 의사 확인
- 실제 출석 체크
- 공지 작성
- 공지 확인률 확인
- 가입 신청 승인/거절
- 초대 링크 생성
- 리마인더 발송

각 플로우는 아래 형식으로 작성한다.

- 목적
- 사용자
- 진입 경로
- 실행 단계
- 성공 결과
- 예외/오류 상황
- 관련 QA 항목

## 회원 앱 플로우

`docs/MEMBER_APP_FLOW.md`에 아래 플로우를 작성한다.

- 휴대폰 인증 로그인
- 가입된 모임 목록 확인
- 모임 전환
- 홈 대시보드 확인
- 일정 참석/불참 응답
- 실제 출석 상태 확인
- 공지 확인
- 공지 좋아요/댓글
- 회비 납부 상태 확인
- 프로필 수정
- 공개 모임 가입 신청
- 비공개 모임 초대코드 가입

각 플로우는 아래 형식으로 작성한다.

- 목적
- 사용자
- 진입 경로
- 실행 단계
- 성공 결과
- 예외/오류 상황
- 관련 QA 항목

## 제약

- 코드 수정 금지
- `docs` 폴더만 수정
- 문서는 한국어로 작성
- 실제 구현 여부와 관계없이 MVP 기준의 기대 동작을 명확히 쓴다.
- 네이버 지도와 실제 결제는 후순위로 둔다.
