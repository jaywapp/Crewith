# Claude 독립 작업: MVP E2E QA 체크리스트 검증

## 목표
Crewith 1차 MVP가 실제 데모 흐름으로 동작하는지 문서 기준으로 점검하고, 빠진 QA 항목을 보강한다.

## 작업 범위
- `docs/DEMO_SCENARIO.md`
- `docs/MVP_RELEASE_CHECKLIST.md`
- `docs/QA_CHECKLIST.md`
- `docs/USER_FLOWS.md`

## 지시
1. 관리자 웹과 회원 앱의 핵심 흐름을 문서끼리 대조한다.
2. 아래 흐름이 체크리스트에 모두 포함되어 있는지 확인한다.
   - 휴대폰 인증 후 회원 앱 진입
   - 여러 모임 가입 상태에서 모임 전환
   - 회원 목록 생성/수정/삭제 및 CSV/TSV 일괄 등록
   - 회비 생성, 납부 상태 변경, 미납 리마인더 발송
   - 일정 생성, 참석 의사 변경, 출석부 저장
   - 공지 작성, 확인 처리, 댓글/좋아요
   - 가입 신청 승인/거절
   - 초대 링크 생성/비활성화/만료 차단
3. 누락된 검증 항목이 있으면 관련 문서에 추가한다.
4. 코드 변경은 하지 말고 문서만 수정한다.

## 완료 기준
- QA 문서에서 위 흐름을 모두 추적할 수 있다.
- 중복 문장은 정리되어 있다.
- 완료 후 이 prompt 파일은 삭제한다.
