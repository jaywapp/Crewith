# Claude Task 01: MVP QA 및 릴리즈 문서 보강

## 목표

Crewith 1차 MVP가 실제 운영 가능한 수준인지 검증할 수 있도록 QA 체크리스트와 릴리즈 체크리스트를 상세화한다.

## 작업 범위

수정 또는 생성할 파일:

- `docs/QA_CHECKLIST.md`
- `docs/MVP_RELEASE_CHECKLIST.md`

## 세부 작업

### 1. `docs/QA_CHECKLIST.md` 상세화

기존 항목을 유지하되 각 QA 항목을 아래 형식으로 확장한다.

- 사전 조건
- 실행 단계
- 기대 결과
- 실패 시 확인할 것

반드시 포함할 영역:

- 관리자 웹
- 회원 앱
- API
- 권한/보안
- 개인정보
- 알림/리마인더
- 릴리즈 게이트

### 2. `docs/MVP_RELEASE_CHECKLIST.md` 신규 작성

아래 영역별 체크리스트를 작성한다.

- 로컬 개발 검증
- API 테스트
- 관리자 웹 빌드
- 회원 앱 테스트
- 수동 QA
- 환경변수 확인
- Firebase 설정 확인
- 보안 규칙 확인
- 개인정보 처리 확인
- 권한 검증
- 배포 전 데이터 초기화
- 배포 후 스모크 테스트
- 롤백 기준

## 제약

- `docs` 폴더만 수정한다.
- `apps`, `packages`, 설정 파일, 빌드 파일은 수정하지 않는다.
- 구현 완료 여부를 단정하지 않는다.
- 현재 구현된 것과 추후 구현이 필요한 것을 명확히 구분한다.
- 문서는 한국어로 작성한다.

## 완료 보고

작업 완료 후 다음을 알려준다.

- 변경한 파일 목록
- 추가한 주요 QA 영역
- 릴리즈 전 반드시 확인해야 할 위험 요소
