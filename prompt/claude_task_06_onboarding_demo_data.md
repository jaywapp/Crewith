# Claude Task 06: MVP 온보딩 및 데모 데이터 시나리오 문서화

## 목표

Crewith 1차 MVP를 처음 보는 사람이 관리자 웹과 회원 앱을 빠르게 이해하고 시연할 수 있도록 온보딩 문서와 데모 시나리오를 작성한다.

## 작업 범위

생성 또는 수정할 파일:

- `docs/DEMO_SCENARIO.md`
- `docs/ONBOARDING.md`
- 필요 시 `docs/README.md`

## 세부 작업

### 1. 데모 시나리오 작성

`docs/DEMO_SCENARIO.md`에 아래 흐름을 작성한다.

- 데모 목적
- 데모 대상 사용자
- 사전 준비
- 관리자 웹 접속
- 대시보드 확인
- 회원 추가
- 회비 생성 및 납부 상태 변경
- 일정 생성 및 참석 상태 확인
- 출석 체크
- 공지 작성 및 확인률 확인
- 가입 신청 승인
- 초대 링크 생성
- 회원 앱 로그인
- 회원 앱에서 일정/공지/회비 확인
- 모임 전환 UI 확인

### 2. 온보딩 문서 작성

`docs/ONBOARDING.md`에 아래 내용을 작성한다.

- 프로젝트 개요
- 저장소 구조
- 관리자 웹 역할
- API 역할
- 회원 앱 역할
- 로컬 실행 순서
- 테스트 실행 순서
- 문서 읽는 순서
- 현재 구현된 것
- 아직 남은 것

### 3. README 연결

필요하면 `docs/README.md`에 새 문서 링크를 추가한다.

## 제약

- 코드 수정 금지
- `docs` 폴더만 수정
- 실제 로컬 명령어는 기존 README 또는 package script에 근거해서 작성한다.
- 확실하지 않은 명령어는 `확인 필요`로 표시한다.
- 문서는 한국어로 작성한다.

## 완료 보고

작업 완료 후 다음을 알려준다.

- 생성/수정한 파일 목록
- 데모 시나리오의 핵심 흐름
- 신규 개발자가 가장 먼저 봐야 할 문서 순서
