# Claude Task 04: API 계약 문서 검토 및 보강

## 목표

Crewith 1차 MVP의 관리자 웹, 회원 앱, API 사이 계약을 문서 기준으로 정리한다. Codex가 API 구현을 수정 중이므로 코드는 절대 수정하지 않는다.

## 작업 범위

수정할 수 있는 파일:

- `docs/API_SPEC.md`
- `docs/DATA_MODEL.md`
- `docs/SCREEN_SPEC.md`

## 세부 작업

### 1. API 엔드포인트 정리

아래 영역별로 엔드포인트를 정리한다.

- 인증
- 관리자 대시보드
- 회원 관리
- 회비 관리
- 일정 관리
- 공지 관리
- 가입 신청
- 초대 링크
- 리마인더
- 회원 앱 홈/상세 데이터

각 엔드포인트마다 다음을 작성한다.

- Method
- Path
- 목적
- 요청 Body
- 응답 Body
- 권한
- 주요 오류

### 2. `clubId` 기준 다중 모임 계약

문서에 아래 내용을 명확히 반영한다.

- 모든 모임 리소스는 `clubId` 기준으로 접근한다.
- 존재하지 않는 `clubId`는 404를 반환해야 한다.
- 회원은 가입된 모임의 데이터만 볼 수 있다.
- 운영진은 본인이 운영진인 모임의 관리자 데이터만 볼 수 있다.
- 한 사용자는 여러 모임에 서로 다른 역할로 가입할 수 있다.

### 3. 화면별 API 매핑

`docs/SCREEN_SPEC.md` 또는 별도 섹션에 아래 화면별 필요한 API를 매핑한다.

- 관리자 대시보드
- 회원 관리
- 회비 관리
- 일정 관리
- 공지 관리
- 가입/초대 관리
- 리마인더
- 회원 앱 홈
- 회원 앱 일정
- 회원 앱 공지
- 회원 앱 회비
- 회원 앱 더보기

## 제약

- `docs` 폴더만 수정한다.
- `apps/api`, `apps/admin-web`, `apps/mobile-app` 코드는 수정하지 않는다.
- 실제 구현과 다른 부분은 `계약상 필요하지만 구현 확인 필요`로 표시한다.
- 문서는 한국어로 작성한다.
